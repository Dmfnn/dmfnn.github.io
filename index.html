    2. Creating manifest.json

    The manifest defines your app's metadata:

    {
  "name": "My App", //Name must be maximum 30 characters long.
  "pkg": "com.example.myapp", //Package name must be maximum 25 characters long.
  "version": "1", //Must be integer.
  "launcherScreen": "main" //Defines, which screen will be launched first.
}

    3. Creating HTML Screens

    HTML files must contain the replacement tag where JavaScript will be injected:

    <!DOCTYPE html>
<html>
<head>
    <title>My App</title>
    <style>
        /* Your styles here */
    </style>
</head>
<body>
    <h1>Hello HOS!</h1>
    <button id="myBtn">Click Me</button>
    
    <script>
        H-R-T-HOS-REPLACEMENT
    </script>
</body>
</html>

    4. Writing JavaScript Logic

    JavaScript files contain your app's logic and API calls:

    // main.js
document.getElementById('myBtn').onclick = function() {
    RuntimeApi.maketoast('Button clicked!');
    
    // Save preference
    PreferenceApi.setpkey('lastClick', new Date().toString());
};

// Read file example
if (PermissionApi.checkSelfPermission('READ_STORAGE')) {
    const content = FileApi.readfile('/storage/test.txt');
    console.log(content);
} else {
    PermissionApi.requestPermission('READ_STORAGE');
}

    ‚ùóRemember: "Screens" in HOS is analogue of Android activity system, every screen is a pair of 1 HTML and 1 JavaScript file with the same name.

    5. Compiling Your App

    Use HOS Shell to compile your project into an HPKG package:

    compilehpkg /storage/emulated/0/projects/your-app

    This creates <pkg>.hpkg in the project directory.

    6. Installing the Package

    Install the compiled package using HOS Shell:

    installhpkg /storage/emulated/0/projects/your-app/<pkg>.hpkg

    Your app will now appear in the app list and can be launched!

    7. Common Shell Commands

    • exec [package] - Launch an application
    • read [path] - Read file contents
    • write [path] [text] - Write to a file
    • mkdir [path] - Create directory
    • plist [path] - List directory contents
    • remove [path] - Delete file or directory

    Tips & Best Practices

    • Always request permissions before using protected APIs
    • Check the system.debug.enableJavaScriptLogs setting, to prevent your app logs can be readed.
    • Use PreferenceApi for app settings and user data
    • Keep assets in the assets/ folder for easy access
    • Test your app thoroughly before distribution
    • Use ScreenApi to navigate between multiple screens
